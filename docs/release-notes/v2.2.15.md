# v2.2.15

## 新功能

- **Telegram 通知支持 Topic 推送** — 通知渠道新增可选字段 `message_thread_id`，可将消息发送到群组的指定话题（Topic），避免不同通知混在一起。留空则发到默认话题，行为不变。

## 修复

- **定时停止编辑入口消失** — 编辑任务时，定时停止的 cron 输入框未渲染，仅显示描述文字。原因是 StopScheduleInput 组件初始化时 `rules` 为空数组，watcher 短路比较导致默认空规则未被创建。
- **Docker 健康检查 404** — `wget --spider` 发送 HEAD 请求在部分环境下经 nginx 代理后返回 404，改用 `curl --fail` 发送 GET 请求。同时在 Gin 日志中跳过 `/api/v1/health` 路径，消除每 30 秒一条的日志噪音。
- **Alpine 镜像 Node.js 版本过旧** — 运行时基础镜像从 `alpine:3.19` 改为 `node:20.19.0-alpine`，解决 sharp 等依赖安装失败（要求 `node >=20.19.0`，旧镜像仅提供 v20.15.1）。
- **任务列表选择框未居中** — 勾选任务时 checkbox 图标偏移，原因是表格单元格水平内边距被清零后选择列未单独补偿居中。
- **低分辨率下列表区域不可见** — Open API 页面改为内容区内滚动，并压缩低高度桌面下的统计卡和说明卡间距，避免创建后的令牌列表被底部说明区域挤出可视范围；同步修复依赖管理在 Android 面具版运行时卡片出现时，依赖列表被前置卡片挤出可视范围的问题。
- **自定义日志背景图片不显示** — 修正日志背景层级顺序，并移除订阅拉取日志、依赖安装日志、任务日志文件浏览器等组件中的硬编码深色背景，避免自定义日志背景图片被渐变或组件背景完全遮住。

## 改进

- **Alpine 依赖安装失败提示** — 在 Alpine 镜像中安装 ddddocr、onnxruntime 等需要 glibc 的依赖时，自动检测并提示用户切换到 Debian 版镜像。
- **任务列表列宽优化** — 缩减任务名称、命令/脚本、定时规则三列的占比，操作列从 210px 加宽到 260px 并增加按钮间距，避免操作按钮过于密集。
- **屏蔽搜索引擎收录** — 三层防护防止面板地址被搜索引擎索引：`robots.txt` 禁止爬取、`X-Robots-Tag` HTTP 响应头、HTML `<meta>` 标签。覆盖 nginx 部署和直连后端两种场景。
- **深色模式 UI 优化** — ECharts 图表适配深色背景（tooltip/网格线/坐标轴/数据点）；修复侧边栏 logo 区域 `color-mix(..., white)` 在暗色下偏亮；仪表盘卡片/面板增加深色边框和阴影深度；主题切换时自动刷新编辑器/日志颜色变量。
- **订阅拉取入口简化** — 订阅列表只保留“按订阅设置拉取”，移除临时选择“覆盖/保留本地”的下拉模式；覆盖策略改由订阅设置里的“覆盖拉取”开关统一控制，手动拉取 API 不再接收临时覆盖模式参数。
