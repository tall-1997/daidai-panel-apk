# v2.2.6

## 重点修复

- 修复飞牛 OS 等第三方 NAS 浏览器登录返回 403 的问题；CORS 中间件对私有/局域网/Loopback IP（192.168.\*、10.\*、172.16-31.\*、127.0.0.1、::1）的 Origin 自动放行，并新增 RFC 7239 `Forwarded` header 支持。公网域名仍走严格白名单。
- 修复部分用户面板内安装 Python 依赖报 `ERROR: Cannot set --home and --prefix together` 的问题；面板内部所有 pip 调用（install / uninstall / list / show）前会主动剥离宿主机预设的 `PIP_PREFIX` / `PIP_HOME` / `PIP_TARGET` / `PIP_ROOT` / `PIP_USER` / `PIP_INSTALL_OPTION` / `PYTHONUSERBASE` 等冲突变量。
- 修复订阅管理私有仓库 Token 鉴权在反代 / CDN / 企业级 GitLab 等环境下拉取失败的问题；改为将 `username:token` 直接嵌入仓库 URL（青龙同款方式），不再依赖 `GIT_HTTP_EXTRA_HEADER`。日志中的仓库地址会自动脱敏 Token。
- 修复 Bark 通知渠道忽略 `context` 中动态跳转链接的问题；脚本/API 调用 `notify.send()` 时若在 context 传入 `url`，Bark 推送会按 context.url > cfg.url 优先级写入请求体。
- 进一步加固订阅拉取的路径处理：`config.Load` 加载阶段统一将数据目录、脚本目录、日志目录规范化为绝对路径，从源头消除相对路径下的目录重复拼接，hook 工作目录 / 单文件下载 / 任务候选扫描全部受益。
- 修复 Docker 容器每次重启会用 `entrypoint.sh` 内嵌模板**覆盖整个** `/app/config.yaml` 的问题；用户在面板内改过的 CORS、信任代理、JWT 等配置在 watchtower 自动升级 / 容器重启后不再丢失。

## 新功能

- Docker 镜像新增 `CORS_ORIGINS` 环境变量，可通过 compose 直接注入公网/反代域名（多个用逗号分隔），无需进入容器手改 yaml。
- Docker 镜像新增 `PUID` / `PGID` 环境变量（LinuxServer.io 风格），Alpine 镜像内置 `su-exec`、Debian 镜像内置 `gosu`，容器以宿主机用户运行后，SMB/NFS 共享下的脚本文件权限与 NAS 账号一致。默认不设保持原有 root 行为。
- 二进制部署新增 `packaging/linux/daidai-panel.service` systemd 单元模板与 `packaging/linux/README.md` 部署指南；模板内置 PIP\_\* 空赋值清理、`User=daidai` 非 root 运行示例、SSE 长连接反代提示等飞牛 OS / 群晖 / 绿联场景常见配置。

## 优化

- Dockerfile（Alpine 与 Debian 双版本）新增 `HEALTHCHECK` 指令，飞牛 OS / 群晖等容器面板可准确反映容器健康状态。
- 容器启动前主动写探测文件预检数据目录可写性，挂载点 owner / SELinux / `:ro` 卷等常见问题会直接输出中文修复指引，不再被 SQLite 的 EACCES 报错淹没。
- 内置 nginx 反代单独抽出 `/api/*/stream` 路径，加 `proxy_http_version 1.1` + `Connection ""` + `X-Accel-Buffering: no` + 24h read timeout，任务实时日志、订阅拉取日志的 SSE 长连接在多层反代下更稳。
- CORS 拒绝跨域请求时输出详细日志（`origin`、`Host`、`X-Forwarded-Host`、`Forwarded` 全部打印）+ 配置引导，单一 origin 5 分钟内只记一次避免刷屏。
