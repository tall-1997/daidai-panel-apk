# v2.2.7

> 紧急修复 v2.2.6 的回归问题。请所有已升级到 v2.2.6 的 Docker 用户尽快升级到 v2.2.7，**数据未丢失，重启容器即可恢复**。

## 重点修复（P0）

- 修复从 v2.2.5 升级到 v2.2.6 后，Docker 面板登录界面回到"全新初始化"、用户/任务/订阅等数据看不到的问题。**数据本身未丢失**（仍在挂载目录 `Dumb-Panel/daidai.db`），升级到 v2.2.7 重启容器后即可自动恢复。

### 根因

镜像构建时 `Dockerfile` 会把仓库内 `server/config.yaml` 复制到容器内 `/app/config.yaml`，这份占位配置用的是相对路径 `./data/daidai.db`。v2.2.5 及更早的 entrypoint 会在每次启动时**强制覆盖**这份占位为 `${DATA_DIR}/daidai.db` 绝对路径，所以一直工作正常。

v2.2.6 把 entrypoint 改成幂等（避免覆盖用户手改过的配置），但忽略了"镜像内置的占位 config 本身就是错的"这种情况——幂等逻辑保留了相对路径的占位配置，导致面板在 cwd 下新建了一个空 DB `/app/data/daidai.db`，旧数据（`/app/Dumb-Panel/daidai.db`）找不到。

### 修复方式

entrypoint 现在会检测 `/app/config.yaml` 是否仍是"镜像默认占位"（含 `./data/daidai.db` 或 `dir: ./data` 相对路径），是则重写为绝对路径配置；用户在面板里手改过的配置（不含上述相对路径特征）继续保留不变。

## 升级路径

- **已升级到 v2.2.6 出现"初始化"界面的用户**：
  - 升级到 v2.2.7（`docker compose pull && docker compose up -d`，或等 watchtower 自动拉取）
  - 容器重启时 entrypoint 会自动识别并重写 `/app/config.yaml`，旧数据立即可见，无需手动操作
- **仍在 v2.2.5 的用户**：可直接升级到 v2.2.7，跳过 v2.2.6
- **二进制部署用户**：本次问题仅影响 Docker 部署，二进制部署不受影响
