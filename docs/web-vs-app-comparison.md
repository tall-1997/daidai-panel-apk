## 三、功能缺失优先级排序

### 高优先级 (P0) - 核心功能缺失
1. **订阅管理** - 完全未实现，是自动化任务的重要功能
2. **仪表盘** - 完全未实现，是用户首次打开的入口页面
3. **个人设置** - 完全未实现，用户无法修改密码等个人信息

### 中优先级 (P1) - 重要功能缺失
1. **用户管理** - 完全未实现，多用户场景必需
2. **定时备份** - 自动备份功能缺失
3. **任务视图** - 自定义筛选视图
4. **脚本版本管理** - 脚本历史版本
5. **文件拖拽移动** - 脚本文件管理体验
6. **值脱敏显示** - 环境变量安全显示

### 低优先级 (P2) - 增强功能缺失
1. **开放 API 管理** - API Key 管理界面
2. **API 文档** - 内置 API 文档查看
3. **自定义通知模板** - 推送模板定制
4. **系统更新** - 面板内一键更新
5. **代理设置** - 网络代理配置
6. **日志统计** - 执行统计分析

## 四、Web API 接口清单（供 App 对接参考）

### 订阅管理 API
- `GET /subscriptions` - 获取订阅列表
- `POST /subscriptions` - 创建订阅
- `PUT /subscriptions/:id` - 更新订阅
- `DELETE /subscriptions/:id` - 删除订阅
- `POST /subscriptions/:id/pull` - 手动拉取
- `GET /subscriptions/:id/logs` - 获取拉取日志
- `GET /subscriptions/settings` - 获取订阅设置
- `PUT /subscriptions/settings` - 更新订阅设置

### 用户管理 API
- `GET /users` - 获取用户列表
- `POST /users` - 创建用户
- `PUT /users/:id` - 更新用户
- `DELETE /users/:id` - 删除用户
- `PUT /users/:id/password` - 修改密码
- `PUT /users/:id/role` - 修改角色

### 仪表盘 API
- `GET /dashboard/stats` - 获取统计数据
- `GET /dashboard/trend` - 获取执行趋势

### 开放 API
- `GET /api-keys` - 获取 API Key 列表
- `POST /api-keys` - 创建 API Key
- `DELETE /api-keys/:id` - 删除 API Key

### 个人设置 API
- `GET /profile` - 获取个人信息
- `PUT /profile` - 更新个人信息
- `PUT /profile/password` - 修改密码
- `POST /profile/2fa/enable` - 启用 2FA
- `POST /profile/2fa/disable` - 禁用 2FA
- `GET /profile/sessions` - 获取会话列表
- `DELETE /profile/sessions/:id` - 终止会话
- `GET /profile/login-logs` - 获取登录日志
